# AGENTS.md ‚Äî Labs64.IO :: API Gateway

Traefik ForwardAuth middleware ‚Äî verifies OIDC/JWT tokens and enforces path-based RBAC. All inbound requests pass through this proxy.

## Ecosystem role

- Deployed as `traefik-authproxy` Helm chart in `labs64io` namespace.
- Traefik forwards every request to `/auth` before routing to upstream services.
- On success, emits the full trusted header contract that upstream services trust: `X-Auth-User`, `X-Auth-Scopes`, `X-Auth-Tenant` (`-` when tenant-less), `X-Request-ID` (echoed when well-formed, otherwise generated as UUIDv7). All four are set on **every** 2xx (empty when not applicable) so Traefik's `authResponseHeaders` always overwrite client-supplied values. Values are sanitized to `^[a-zA-Z0-9_.:-]+$` (CR/LF stripped) ‚Äî keep identical to the `auth-context` libraries in `labs64.io-commons`.

## Critical guardrails

1. **Never hardcode credentials** ‚Äî env vars or K8s Secrets only.
2. **Preserve `l64user`** (uid/gid 1064) in Dockerfiles.
3. **Auth-policy scopes must be consistent with OIDC provider claims** (`TOKEN_SCOPES_CLAIM_PATHS`).

## Auth proxy details

- **App layout** (no longer single-file): `traefik_authproxy.py` (FastAPI app, `/auth` decision, JWT verification) + `policy_store.py` (in-memory auth-policy table, OpenAPI-template matching, `parse_cedar_document`/`parse_policy_document`) + `policy_sync.py` (k8s Service discovery + per-module `auth-policy.cedar` fetch ‚Äî routing table and Cedar decision input in one file) + `policy_bundle.py` (signed-bundle policy source, still paired `auth-policy.json` + `auth-policy.cedar`) + `cedar_edge.py` (in-process Cedar edge PDP).
- **JWT verification**: `python-jose` with RS256. JWKS from OIDC provider via discovery URL.
- **Scope extraction**: `TOKEN_SCOPES_CLAIM_PATHS` env var (comma-separated dot-paths, supports `{audience}` placeholder); union of the `scope` claim and Keycloak-style role claims.
- **Path matching**: per-operation OpenAPI template matching, sourced differently per policy source. Live discovery: from each module's `/.well-known/auth-policy.cedar` ‚Äî the @path/@method/@public/@tenantRequired/@scopes annotations the commons `OpenApiAuthPreprocessor` stamps on every generated permit (`policy_store.parse_cedar_document`); `auth-policy.json` is no longer fetched on this path. Signed bundle: still from the bundle's `modules/<name>.json` (`policy_store.parse_policy_document`) ‚Äî unchanged pending a bundle-format update. Plus static prefix policies (`STATIC_POLICY_FILE`) in both cases. Module routes take priority; static prefixes (longest-prefix) are consulted only when no route template matches; no match at all fails closed (403).
- **Hot reload**: `POST /reload` reloads the static policy file and triggers an immediate module auth-policy re-sync, without restart.
- **Readiness**: `GET /health/ready` returns 503 until the first module auth-policy sync pass has completed (`PolicySync.ready()`); `GET /health` is the liveness probe.
- **JWKS caching**: TTL-based (`JWKS_CACHE_TTL`, default 3600s). Prefetched on startup.
- **Cedar edge tier**: `CEDAR_MODE=off|shadow|enforce` (default `shadow`) ‚Äî the declarative legacy-decision ‚Üî Cedar-decision switch. On live discovery, fetching `/.well-known/auth-policy.cedar` is unconditional (it's the routing source too, regardless of mode); `CEDAR_MODE` only gates whether the engine is loaded and consulted for the *decision*. The generated per-operation Cedar policies arrive over whichever policy source is active: inside the signed bundle (`modules/<name>.cedar`) or, under live discovery, fetched from each module's `/.well-known/auth-policy.cedar` (build-generated by the commons `OpenApiAuthPreprocessorCli`; validated before accept, last-known-good on failure ‚Äî a 404/fetch failure is now treated as `invalid`/`failed` like a bad routing document, since routing depends on it). Modules are keyed by the first segment of their gateway base path (e.g. `/payment-gateway/api/v1` ‚Üí `payment-gateway`) so Cedar resource ids agree across build, bundle, and live discovery. `shadow` logs `cedar-shadow ... match=True|False` per module-route request without changing behavior; `enforce` makes Cedar the decision for module routes (legacy public/tenant/scope logic retired there ‚Äî static prefixes stay legacy). Engine errors and unknown operations fail closed.

  #### Enforcement logging (testing phase)

  Every module-route Cedar evaluation emits a **summary** line on `traefik_authproxy`:

  ```
  cedar-<mode> outcome=<enforced|shadow>-<allow|deny> module=<m> op=<operationId> \
    decision=<allow|deny|error> legacy=<allow|deny> match=<bool> reasons=<policyIds|-> requestId=<id> ‚Äî <METHOD> <path>
  ```

  Level: INFO for a clean allow, WARN for deny/error/mismatch. In enforce mode this line **is** the block record ‚Äî it carries the matched policy `reasons`.

  Sensitive fields (user, tenant, scopes, resource) are **not** in the summary. To see them during the Cedar testing phase, raise only the dedicated child logger:

  ```python
  logging.getLogger("traefik_authproxy.cedar.detail").setLevel(logging.DEBUG)
  ```

  It emits `cedar-detail requestId=‚Ä¶ user=‚Ä¶ tenant=‚Ä¶ scopes=‚Ä¶ resource=‚Ä¶::‚Ä¶[ error=‚Ä¶] ‚Äî <METHOD> <path>`, joinable to the summary by `requestId`. Leave it above DEBUG in shared/staging logs.

### Environment variables

| Variable | Default | Description |
|----------|---------|-------------|
| `OIDC_URL` | `http://mock-oidc.tools.svc.cluster.local:8080` | OIDC provider base URL |
| `OIDC_REALM` | `default` | OIDC realm |
| `OIDC_AUDIENCE` | `account` | Expected JWT audience |
| `TOKEN_SCOPES_CLAIM_PATHS` | `scope,realm_access.roles,resource_access.{audience}.roles` | JWT claim paths for scopes |
| `TOKEN_TENANT_CLAIM_PATH` | `tenant` | JWT dot-path for the tenant identifier (`X-Auth-Tenant`) |
| `STATIC_POLICY_FILE` | `static_policies.yaml` | Static prefix policies for non-OpenAPI surfaces (UI bundles) |
| `POLICY_REFRESH_INTERVAL` | `30` | Periodic re-fetch interval for module auth policies (seconds) |
| `POD_NAMESPACE` | (from serviceaccount namespace file, else `default`) | Namespace searched for `labs64.io/auth-policy=true` labeled Services |
| `JWKS_CACHE_TTL` | `3600` | JWKS cache TTL (seconds) |
| `POLICY_BUNDLE_DIR` | (unset) | Signed policy bundle dir. When set, live discovery is disabled |
| `CEDAR_MODE` | `shadow` | Cedar edge tier: `off` / `shadow` (log diff) / `enforce` (Cedar decides module routes). Works under both policy sources (bundle or live discovery) |

## Build, run, test

```bash
cd traefik-authproxy
just docker         # build + push to localhost:5005
just run            # build + run with local OIDC config
just docu           # open ReDoc + Swagger docs
```

Tests: `pytest` in `tests/`. Local docs: `:8081/docs`.
