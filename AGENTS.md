# AGENTS.md — Labs64.IO :: API Gateway

Traefik ForwardAuth middleware — verifies OIDC/JWT tokens and enforces path-based RBAC. All inbound requests pass through this proxy.

## Ecosystem role

- Deployed as `traefik-authproxy` Helm chart in `labs64io` namespace.
- Traefik forwards every request to `/auth` before routing to upstream services.
- On success, emits the full trusted header contract that upstream services trust: `X-Auth-User`, `X-Auth-Scopes`, `X-Auth-Tenant` (`-` when tenant-less), `X-Request-ID` (echoed when well-formed, otherwise generated as UUIDv7). All four are set on **every** 2xx (empty when not applicable) so Traefik's `authResponseHeaders` always overwrite client-supplied values. Values are sanitized to `^[a-zA-Z0-9_.:-]+$` (CR/LF stripped) — keep identical to the `auth-context` libraries in `labs64.io-commons`.

## Critical guardrails

1. **Never hardcode credentials** — env vars or K8s Secrets only.
2. **Preserve `l64user`** (uid/gid 1064) in Dockerfiles.
3. **Auth-policy scopes must be consistent with OIDC provider claims** (`TOKEN_SCOPES_CLAIM_PATHS`).

## Auth proxy details

- **App layout** (no longer single-file): `traefik_authproxy.py` (FastAPI app, `/auth` decision, JWT verification) + `policy_store.py` (in-memory auth-policy table, OpenAPI-template matching, `parse_cedar_document`) + `policy_sync.py` (k8s Service discovery + per-module `auth-policy.cedar` fetch — routing table and Cedar decision input in one file) + `policy_bundle.py` (signed-bundle policy source, `auth-policy.cedar`) + `cedar_edge.py` (in-process Cedar edge PDP).
- **JWT verification**: `python-jose` with RS256. JWKS from OIDC provider via discovery URL.
- **Scope extraction**: `TOKEN_SCOPES_CLAIM_PATHS` env var (comma-separated dot-paths, supports `{audience}` placeholder); union of the `scope` claim and Keycloak-style role claims.
- **Path matching**: per-operation OpenAPI template matching, sourced differently per policy source. Live discovery: from each module's `/.well-known/auth-policy.cedar` — the @path/@method/@public/@tenantRequired/@scopes annotations the commons `OpenApiAuthPreprocessor` stamps on every generated permit (`policy_store.parse_cedar_document`). Signed bundle: from the bundle's `modules/<name>.cedar`. Plus static prefix policies (`STATIC_POLICY_FILE`) in both cases. Module routes take priority; static prefixes (longest-prefix) are consulted only when no route template matches; no match at all fails closed (403).
- **Hot reload**: `POST /reload` reloads the static policy file and triggers an immediate module auth-policy re-sync, without restart.
- **Readiness**: `GET /health/ready` returns 503 until the first module auth-policy sync pass has completed (`PolicySync.ready()`); `GET /health` is the liveness probe.
- **JWKS caching**: TTL-based (`JWKS_CACHE_TTL`, default 3600s). Prefetched on startup.
- **Cedar edge tier**: Cedar IS the decision for module routes. The generated per-operation Cedar policies arrive over whichever policy source is active: inside the signed bundle (`modules/<name>.cedar`) or, under live discovery, fetched from each module's `/.well-known/auth-policy.cedar` (build-generated by the commons `OpenApiAuthPreprocessorCli`; validated before accept, last-known-good on failure). Modules are keyed by the first segment of their gateway base path (e.g. `/payment-gateway/api/v1` → `payment-gateway`) so Cedar resource ids agree across build, bundle, and live discovery. Engine errors and unknown operations fail closed.

  #### Enforcement logging

  Every module-route Cedar evaluation emits a **summary** line on `traefik_authproxy`:

  ```
  cedar outcome=enforced-<allow|deny> module=<m> op=<operationId> \
    decision=<allow|deny|error> reasons=<policyIds|-> requestId=<id> — <METHOD> <path>
  ```

  Level: INFO for a clean allow, WARN for deny/error. This line carries the matched policy `reasons`.

  Sensitive fields (user, tenant, scopes, resource) are **not** in the summary. To see them during the Cedar testing phase, raise only the dedicated child logger:

  ```python
  logging.getLogger("traefik_authproxy.cedar.detail").setLevel(logging.DEBUG)
  ```

  It emits `cedar-detail requestId=… user=… tenant=… scopes=… resource=…::…[ error=…] — <METHOD> <path>`, joinable to the summary by `requestId`. Leave it above DEBUG in shared/staging logs.

### Environment variables

| Variable | Default | Description |
|----------|---------|-------------|
| `OIDC_URL` | `http://mock-oidc.tools.svc.cluster.local:8080` | OIDC provider base URL |
| `OIDC_REALM` | `default` | OIDC realm |
| `OIDC_AUDIENCE` | `account` | Expected JWT audience |
| `TOKEN_SCOPES_CLAIM_PATHS` | `scope,realm_access.roles,resource_access.{audience}.roles` | JWT claim paths for scopes |
| `TOKEN_TENANT_CLAIM_PATH` | `tenant` | JWT dot-path for the tenant identifier (`X-Auth-Tenant`) |
| `STATIC_POLICY_FILE` | `static_policies.yaml` | Static prefix policies for non-OpenAPI surfaces (UI bundles) |
| `POLICY_REFRESH_INTERVAL` | `30` | Periodic re-fetch interval for module auth policies (seconds) |
| `POD_NAMESPACE` | (from serviceaccount namespace file, else `default`) | Namespace searched for `labs64.io/auth-policy=true` labeled Services |
| `JWKS_CACHE_TTL` | `3600` | JWKS cache TTL (seconds) |
| `POLICY_BUNDLE_DIR` | (unset) | Signed policy bundle dir. When set, live discovery is disabled |


## Build, run, test

```bash
cd traefik-authproxy
just docker         # build + push to localhost:5005
just run            # build + run with local OIDC config
just docu           # open ReDoc + Swagger docs
```

Tests: `pytest` in `tests/`. Local docs: `:8081/docs`.
